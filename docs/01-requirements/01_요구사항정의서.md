# 요구사항 정의서

## 1. 문서 정보

* **문서 목적**: 동네책장 플랫폼의 요구사항을 명확히 정의하여 이해관계자 간 공통 이해를 확보한다.
* **대상**: 기획, 디자인, 프론트엔드
* **범위**: 웹 기반 중고책 거래 및 독서 모임 커뮤니티 플랫폼
* **작성일**: 2026-01-20
* **버전**: 1.0

---

## 2. 시스템 개요

동네책장은 사용자가 중고책을 등록·조회·검색하고, GPS 기반 위치 서비스를 통해 근처의 책을 찾아 거래할 수 있는 플랫폼이다. 또한 독서 모임 게시판을 통해 모임을 모집하고, 거래 후기를 작성·조회하여 신뢰성 있는 거래 환경을 제공한다. 로그인 사용자만 책 등록, 모임 글 작성, 후기 작성 등의 기능을 이용할 수 있으며, 비회원은 조회 및 검색만 가능하다.

---

## 3. 사용자 유형

* **비회원**: 책 목록 조회, 검색, 모임 게시판 조회만 가능
* **일반회원**: 책 등록/수정/삭제, 찜하기, 모임 글 작성/댓글, 후기 작성/조회, 실시간 채팅, 프로필 관리 가능

---

## 4. 기능 요구사항

| ID | 우선순위 | 요구사항 |
|---|---|---|
| FR-01 | 1순위 | 사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다. |
| FR-02 | 1순위 | 사용자는 책 제목, 저자, 카테고리, 상태 등을 입력하여 중고책을 등록할 수 있어야 한다. |
| FR-03 | 1순위 | 사용자는 등록된 책 목록을 조회할 수 있어야 한다. |
| FR-04 | 1순위 | 사용자는 책 제목, 저자 등으로 검색할 수 있어야 한다. |
| FR-05 | 1순위 | 사용자는 책 상세 정보를 조회할 수 있어야 한다. |
| FR-06 | 2순위 | 사용자는 이메일과 비밀번호로 회원가입할 수 있어야 한다. |
| FR-07 | 2순위 | 로그인 사용자는 자신이 등록한 책을 수정할 수 있어야 한다. |
| FR-08 | 2순위 | 로그인 사용자는 자신이 등록한 책을 삭제할 수 있어야 한다. |
| FR-09 | 2순위 | 로그인 사용자는 관심 있는 책을 찜할 수 있어야 한다. |
| FR-10 | 2순위 | 사용자는 GPS 기반으로 근처 위치의 책을 검색할 수 있어야 한다. |
| FR-11 | 2순위 | 사용자는 독서 모임 게시판에서 모임 모집 글을 조회할 수 있어야 한다. |
| FR-12 | 2순위 | 로그인 사용자는 독서 모임 게시판에 글을 작성할 수 있어야 한다. |
| FR-13 | 2순위 | 로그인 사용자는 모임 게시글에 댓글을 작성할 수 있어야 한다. |
| FR-14 | 3순위 | 로그인 사용자는 거래 후기를 작성할 수 있어야 한다. |
| FR-15 | 3순위 | 사용자는 특정 회원이 작성한 후기를 조회할 수 있어야 한다. |
| FR-16 | 3순위 | 사용자는 특정 회원이 받은 후기를 조회할 수 있어야 한다. |
| FR-17 | 3순위 | 로그인 사용자는 프로필 정보를 수정할 수 있어야 한다. |
| FR-18 | 3순위 | 사용자는 알림을 통해 채팅, 댓글 알림 등을 받을 수 있어야 한다. |

## 5. 비기능 요구사항

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-01 | 필수 | 모든 페이지는 모바일 우선 반응형 디자인으로 제공되어야 한다. |
| NFR-02 | 필수 | 페이지 초기 로딩 시간은 3초 이내여야 한다. |
| NFR-03 | 필수 | UI/UX는 직관적이고 사용하기 쉬워야 한다(Figma 디자인 기준). |
| NFR-04 | 권장 | 주요 사용자 액션(검색, 페이지 이동)은 1초 이내에 반응해야 한다. |
| NFR-05 | 권장 | 크로스 브라우저 호환성을 지원해야 한다(Chrome, Safari, Edge 등). |
| NFR-06 | 권장 | 이미지는 최적화되어 빠르게 로드되어야 한다. |

---

## 6. 제약 및 가정

* 브라우저 기반 웹 앱으로 제공한다.
* React, Next.js, Zustand, Tailwind CSS 기술 스택을 사용한다.
* GPS 기반 위치 서비스는 브라우저의 Geolocation API를 사용한다.
* 모바일 우선(Mobile-first) 디자인 원칙을 따른다.
